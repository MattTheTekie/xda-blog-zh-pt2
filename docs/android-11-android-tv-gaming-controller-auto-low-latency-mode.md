# Android 11 将带着更好的游戏控制器支持来到 Android TV

> 原文：<https://www.xda-developers.com/android-11-android-tv-gaming-controller-auto-low-latency-mode/>

经过一轮全面的测试后，Android 11 [已经面向智能手机](https://www.xda-developers.com/android-11-stable-google-pixel-oneplus-xiaomi-realme-oppo/)发布。[的源代码现在已经可用](https://www.xda-developers.com/android-11-source-code-aosp/)，新的软件现在正在[慢慢推出智能手机](https://www.xda-developers.com/android-11-update-tracker/)。其他基于 Android 的平台，如 Android TV，迟早会尝到最新版本的滋味。继智能手机发布之后，谷歌推出了适用于 Android 电视设备的 Android 11，并进行了一些关键改进，这些改进对于改善电视用户体验至关重要。

你将在这个版本中看到的大多数改进，至少在 Android TV 方面，并不是真正的功能更新，而是底层平台的改进。[官方公告博客文章](https://android-developers.googleblog.com/2020/09/introducing-android-11-on-android-tv.html)提到了几项性能改进，如增强的内存管理，以及一些隐私功能，如一次性权限等。这些改进大部分也存在于智能手机的 Android 11 中。然而，一些改进专门改善了 Android 电视设备的体验。

例如，我们有更好的游戏控制器支持。从 Android 11 开始，Android 电视设备通过蓝牙和 USB 原生支持[任天堂 Switch Pro 控制器](https://www.xda-developers.com/android-11-google-pixel-improves-nintendo-switch-joy-con-pro-controller-support/)，它们还将通过 USB 支持 Steam 控制器。这种[也适用于智能手机](https://www.xda-developers.com/android-11-84-new-gaming-controller-mappings/)，但在电视的情况下，这种控制器支持对于玩游戏更为重要，因为你没有任何触摸输入可以依靠。此外，Android TV 的 Android 11 还支持 HDMI 2.1 规范引入的自动低延迟模式，该模式可以关闭图形后处理，因此大大减少了延迟。我们还支持低延迟媒体解码、新的调谐器框架和 HDMI CEC HAL 实现的扩展。

从今天开始，Android 11 更新将推出 ADT-3 开发者套件([也有资格参加开发者预览](https://www.xda-developers.com/android-11-developer-preview-adt-3-android-tv/))，这样开发者就可以为其他设备准备好他们的应用程序。至于在消费设备上的可用性，更新时间表是由原始设备制造商决定的，所以你必须联系你的制造商，以了解更新是否会出现在你的电视或机顶盒上。
