# 谷歌为 Android 发布了新地图 SDK 的测试版

对于想尝试谷歌地图 SDK 早期版本的开发者来说，现在有了一个新的测试版。我们被告知这将带来更好的性能和更多。

谷歌 I/O 2019 的主要基调可能已经结束，但开发者大会将持续到今天结束。在这次活动中，我们了解了[许多关于新硬件产品](https://www.xda-developers.com/google-pixel-3a-3a-xl-specs-features-price/)、未来将带给用户的新[服务](https://www.xda-developers.com/arcore-1-9-lg-g8-lg-v50-scene-viewer/)等等。本周，谷歌 Android 版地图 SDK 发布了一些重大消息。对于想要尝试 SDK 早期版本的开发人员来说，现在有了一个新的测试版。我们被告知，与目前的 SDK 相比，这将为应用程序带来更好的性能和支持。

应用程序开发人员利用版本工具为您带来他们在工作中提供的功能。对于想要访问谷歌地图数据的应用程序来说，[他们使用 Android 版的地图 SDK](https://developers.google.com/maps/documentation/android-sdk/intro)。除非你是一名开发人员，否则你可能不知道 Android 版地图 SDK 并不完全建立在与谷歌地图移动应用相同的基础设施上。这导致了一些问题，包括对性能的影响，引入新功能所需的时间，以及最终用户整体更好的稳定性。

开发者已经就这个问题向谷歌反馈了一段时间，事情正在朝着正确的方向发展。山景城科技巨头的工程师们已经工作了几个月，以重构和改进 Android 地图 SDK 的新测试版。除了性能改进和更快地访问新功能之外，这个新的测试版 SDK 应该可以使最终用户的地图相关数据消耗减少 60%。这个测试版也致力于提高稳定性，我们被告知，它包括对超过 65 个开发者社区报告的主要问题的修复。

希望试用这个新的测试版 SDK 的开发人员被鼓励查看他们的迁移指南，因为它将带你完成整个过程。不过请记住，这是一个测试版 SDK，所以请在这里直接向谷歌提交[你可能发现的任何 bug。](https://goo.gle/gmp-android-sdk-beta-issues)
