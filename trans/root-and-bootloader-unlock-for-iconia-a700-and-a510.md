# Iconia A700 和 A510 的 Root 和引导加载程序解锁

> 原文：<https://www.xda-developers.com/root-and-bootloader-unlock-for-iconia-a700-and-a510/>

在发布新设备时，许多开发人员承担的任务是找到设备的根并解锁引导加载程序。在这之后，通常会出现大量定制 rom、内核和其他修改和调整形式的开发。可以说，直到保修彻底失效，设备的生命才真正开始。对于 Acer Iconia A700 和 A510 用户来说，这一旅程现在就可以开始。

Iconia A700 引导加载程序解锁和 root 教程是 XDA 论坛成员 [ZeroNull](http://forum.xda-developers.com/member.php?u=4479985) 和 [alx5962](http://forum.xda-developers.com/member.php?u=501404) 的共同努力，而 A510 教程则由 XDA 论坛成员 [BENETNATH](http://forum.xda-developers.com/member.php?u=3458721) 负责。

对于 A700，bootloader 解锁和 root 相当简单。对于解锁，使用*快速启动 oem 解锁*方法。要是在其他流行设备上也这么简单就好了。获取 root 只是稍微复杂一点，因为用户必须下载并解压缩文件，然后使用管理员权限通过 Windows 命令提示符刷新文件。就是这样。

目前，A510 正在使用 [Iconia A100](http://www.xda-developers.com/tag/acer-iconia-a100/) 漏洞来获得 root。虽然这种方法预计不会持续到下一次固件更新之后，但已经证实它可以在 A510 的所有当前固件版本上工作。解锁引导程序仍然很容易，因为这是相同的*快速启动 oem 解锁*方法。[其他一些公司](http://www.xda-developers.com/android/motorola-at-it-again-locks-down-atrix-4g-even-more/)可以从宏碁那里学到一些经验。

你可以在 [A700 引导程序解锁线程](http://forum.xda-developers.com/showthread.php?t=1710755)和 [A700 根线程](http://forum.xda-developers.com/showthread.php?t=1727105)中找到 A700 指南。A510 用户需要的一切都可以在 [A510 根和引导装载线程](http://forum.xda-developers.com/showthread.php?t=1732888)中找到。
