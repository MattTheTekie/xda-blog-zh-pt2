# 以下是 Android 11 的新表情符号，你现在就可以安装[Root]

Android 11 支持 Unicode 13 的 62 个新表情符号。您可以使用 Magisk 模块在旧版本的 Android 上安装新的表情符号！

今年早些时候发布了 Unicode 13，带来了 62 个新表情符号(117 个，如果你单独计算变化的话)。虽然表面上它们的存在看起来没什么大不了的，但是这些图形资产已经成为在互联网上和通过我们的设备表达的不可或缺的一部分。他们在纯粹的文字中加入情感、意图和性格。新表情符号的频繁出现证明了它们在现代交流和表达中的重要性。那么，得知 Unicode 13 中添加的 62 个表情符号被[添加到 Android 11 Beta 1](https://blog.emojipedia.org/new-emojis-in-android-11-beta/) 中也就不足为奇了。你可以在支持的设备上安装 Android 11 Beta 1 来访问这些表情符号。但也有一种更简单的方法在你现有的软件上安装和体验这些表情符号，假设你有 root 用户。

**[安卓 11 XDA 新闻](https://www.xda-developers.com/tag/android-11/)**

以 Gboard 主题闻名的 XDA 资深成员 [RKBD](https://forum.xda-developers.com/member.php?u=7544065) 已经从一个未发布的 Android 11 版本中获得了包含新表情符号的字体文件。这个字体文件已经被打包到 Magisk 模块中，安装了 Magisk 的用户可以简单地更新他们现有的表情符号。对于大多数 Android 设备，该模块取代了*/系统/字体*中的“ *NotoColorEmoji.ttf* ”文件。在三星设备上，该模块在同一位置替换“ *SamsungColorEmoji.ttf* ”文件。

要记住的一点是，表情符号风格在你的设备上是可见的。像脸书、WhatsApp、Telegram、Discord 和其他一些应用程序都在使用自己的风格，还没有包括新的表情符号。如果该平台不支持新的表情符号，它们将无法在接收者端正确呈现。所以，在你把 [U+1F972](https://emojipedia.org/emoji/%F0%9F%A5%B2/) 发给每个人之前，一定要记住这一点。但是现在安装新的已经变得非常容易，我们应该会看到跨生态系统的更广泛的扩散。Gboard 整合新版本只是时间问题[。](https://www.xda-developers.com/gboard-9-5-3-reveals-minis-feature-chopping-block/)
